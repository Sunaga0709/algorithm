# コムソート

## 概要
コムソートは、離れた要素どうしを比較しながら間隔 `gap` を徐々に縮めるソートです。大きな逆転を先に減らせるのが利点です。`sort/comb` はその基本を学ぶ場所です。

## 使いどころ
- バブルソートの改善例を見たいとき
- 遠い位置の乱れを先に直したいとき
- 比較と交換の流れを追いたいとき

## 手順
1. `gap` を配列長から始める。
2. `gap` を少しずつ縮める。
3. `gap` 離れた 2 要素を比べる。
4. 逆なら交換する。
5. `gap` が 1 になるまで繰り返す。
6. `gap = 1` になった後は、交換がなくなるまで続ける。

## 変化の例
`[5, 1, 4, 2]` を考える。

```text
gap=3: [2, 1, 4, 5]
gap=2: [2, 1, 4, 5]
gap=1: [1, 2, 4, 5]
```

## 実装ポイント
- `gap` は 1 未満にならないようにする。
- 最後は `gap=1` の総仕上げが必要。
- バブルソートより無駄な比較が減ることがある。
- `gap` の縮め方で挙動が変わる。

## 計算量
| 操作 | 計算量 |
| --- | --- |
| 平均 | O(n^2) に近い |
| 最悪 | O(n^2) |
| 追加メモリ | O(1) |
