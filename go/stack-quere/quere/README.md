# キュー

## 概要

`quere` は typo ですが、ここではキューを表します。キューは先入れ先出し、つまり最初に入れた要素を最初に出す構造です。待ち行列のように扱います。

## 使いどころ

- 受付順に処理したいとき
- 幅優先探索のように層ごとに処理したいとき
- 仕事の順番を保ちたいとき

## 手順

1. `enqueue` で末尾に要素を入れる。
2. `dequeue` で先頭から要素を取り出す。
3. 空のときは取り出しをしない。
4. 先頭と末尾を別々に管理すると高速になる。

## 変化の例

初期状態:

```text
front -> [A] [B] [C] <- rear
```

`enqueue(D)`:

```text
front -> [A] [B] [C] [D] <- rear
```

`dequeue()`:

```text
front -> [B] [C] [D] <- rear
```

## 実装ポイント

- 配列で作るなら、先頭削除のずれをどう処理するかが大切。
- 連結リストなら、先頭と末尾の更新がしやすい。
- `front` と `rear` の両方を持つと O(1) で操作しやすい。
- 空判定は `front == nil` のように見る。

## 計算量

| 操作     | 計算量 |
| -------- | ------ |
| enqueue  | O(1)   |
| dequeue  | O(1)   |
| 先頭参照 | O(1)   |
| 検索     | O(n)   |
