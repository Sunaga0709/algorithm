# 二分探索木

## 概要
二分探索木は、各ノードより小さい値を左、大きい値を右に置く木構造です。探す、入れる、消す処理を木の形で行います。`tree/binary-search` はその基本を学ぶ場所です。

## 使いどころ
- 値の大小順を保ちながら扱いたいとき
- 探索を平均的に速くしたいとき
- 順序つきの集合を表したいとき

## 手順
1. 木が空なら、新しい値を根にする。
2. 挿入値が現在ノードより小さければ左へ進む。
3. 大きければ右へ進む。
4. 空き位置に着いたら新ノードを置く。
5. 検索も同じ規則で進む。
6. 削除は、子の数に応じてつなぎ替える。

## 変化の例
`50, 30, 70, 20, 40, 60, 80` を順に入れる。

```text
        [50]
       /    \
    [30]    [70]
    /  \    /  \
 [20] [40] [60] [80]
```

`40` を探す流れ:
```text
50 -> left
30 -> right
40 -> found
```

## 実装ポイント
- ノードは `value`, `left`, `right` を持つ。
- 重複値をどう扱うかを先に決める。
- 削除は、葉、片子、両子で分けると整理しやすい。
- 中順走査をすると昇順で取り出せる。

## 計算量
| 操作 | 平均 | 最悪 |
| --- | --- | --- |
| 挿入 | O(log n) | O(n) |
| 検索 | O(log n) | O(n) |
| 削除 | O(log n) | O(n) |
