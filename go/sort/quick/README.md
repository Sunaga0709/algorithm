# クイックソート

## 概要

クイックソートは、基準値 `pivot` を決めて、左に小さい値、右に大きい値を分け、同じ処理を再帰で繰り返すソートです。平均的に速いのが強みです。`sort/quick` はその基本を学ぶ場所です。

## 使いどころ

- 平均性能を重視したいとき
- インプレースで整列したいとき
- 分割統治の流れを学びたいとき

## 手順

1. `pivot` を 1 つ選ぶ。
2. 配列を `pivot` より小さい側と大きい側に分ける。
3. 左右の部分配列を再帰的に整列する。
4. 最後に結合された順が整列結果になる。

## 変化の例

`[5, 2, 4, 1, 3]` に対して `pivot = 4` とする。

```text
before: [5, 2, 4, 1, 3]

left  : [2, 1, 3]
pivot : [4]
right : [5]

after : [2, 1, 3, 4, 5]
```

## 実装ポイント

- `pivot` の選び方で性能が変わる。
- 端に偏ると最悪計算量が悪化する。
- その場で分割するか、別配列で分けるかを決める。
- 再帰の終了条件を明確にする。

## 計算量

| 操作           | 平均          | 最悪            |
| -------------- | ------------- | --------------- |
| クイックソート | O(n log n)    | O(n^2)          |
| 追加メモリ     | O(log n) 前後 | O(n) 実装による |
