# ヒープソートと TimSort の比較

## 概要

`heap-tim` は名前の通り、ヒープソートと TimSort の学習用ディレクトリです。どちらも整列アルゴリズムですが、考え方がかなり違います。ヒープは木の性質を使い、TimSort は部分的に整列済みの区間を活かします。

## 使いどころ

- ヒープソートを学びたいとき
- TimSort の発想を知りたいとき
- 実装前に両者の違いを整理したいとき

## 手順

### ヒープソート

1. 配列からヒープを作る。
2. 根の最大値または最小値を末尾へ移す。
3. ヒープのサイズを 1 つ減らす。
4. 根から下へ調整する。
5. これを繰り返す。

### TimSort

1. 配列の中の整列済み区間を探す。
2. 小さい区間は挿入ソートで整える。
3. 区間同士を安定にマージする。
4. 全体が 1 本になるまで続ける。

## 変化の例

`[5, 2, 4, 1]` を考える。

```text
ヒープ化: [5, 2, 4, 1]
取り出し: [4, 2, 1, 5]
次の取り出し: [2, 1, 4, 5]
最後: [1, 2, 4, 5]
```

TimSort では、最初から短い整列済み区間を見つけて結合する。

## 実装ポイント

- ヒープソートは最悪計算量が安定している。
- TimSort は実データの局所的な整列を活かしやすい。
- どちらも「部分を整えて全体を作る」発想がある。
- 比較対象の性質が違うので、用途も分けて考える。

## 計算量

| 手法         | 平均       | 最悪       | 追加メモリ |
| ------------ | ---------- | ---------- | ---------- |
| ヒープソート | O(n log n) | O(n log n) | O(1)       |
| TimSort      | O(n log n) | O(n log n) | O(n)       |
