# 双方向連結リスト

## 概要
双方向連結リストは、各ノードが `prev` と `next` を持つ連結リストです。前後どちらにもたどれるので、途中の要素の削除や挿入がしやすいです。`link-list/duplex` はその基本動作を学ぶための場所です。

## 使いどころ
- 途中の要素を頻繁に挿入、削除したいとき
- 先頭からも末尾からもたどりたいとき
- 配列のように連続したメモリを前提にしたくないとき

## 手順
1. ノードに `value`, `prev`, `next` を持たせる。
2. リストは `head` と `tail` を持つ。
3. 末尾へ追加するときは、旧 `tail.next` を新ノードに向ける。
4. 新ノードの `prev` を旧 `tail` に向ける。
5. `tail` を新ノードに更新する。
6. 削除するときは、前後のノード同士をつなぎ直す。

## 変化の例
初期状態:
```text
head -> [10] <-> [20] <-> [30] <- tail
```

`20` のあとに `25` を入れる:
```text
head -> [10] <-> [20] <-> [25] <-> [30] <- tail
```

`20` を消す:
```text
head -> [10] <-> [25] <-> [30] <- tail
```

## 実装ポイント
- `nil` を先頭と末尾の端点として扱う。
- 先頭や末尾の挿入、削除は特別扱いが必要。
- 中央の削除は、`prev.next` と `next.prev` を両方更新する。
- 逆方向走査を使うと、末尾側からの探索が楽になる。

## 計算量
| 操作 | 計算量 |
| --- | --- |
| 先頭挿入 | O(1) |
| 末尾挿入 | O(1) |
| 途中削除 | O(1) |
| 検索 | O(n) |
