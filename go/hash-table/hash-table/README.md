# ハッシュテーブル

## 概要
ハッシュテーブルは、キーをハッシュ関数で配列の添字に変換して値を保存する仕組みです。平均すると高速に検索できます。`hash-table/hash-table` は、その基本構造を学ぶ場所です。

## 使いどころ
- キーから値をすぐ引きたいとき
- 重複判定を速くしたいとき
- 順番よりも検索速度を重視したいとき

## 手順
1. キーをハッシュ関数に通して添字を作る。
2. 添字の位置に値を置く。
3. 同じ添字に複数のキーが来たら衝突を処理する。
4. 検索時は同じハッシュ関数で位置を決める。
5. その位置から目的のキーを探す。

## 変化の例
`key = 12, 22, 32` を `index = key % 10` で入れるとする。

```text
table
0: -
1: -
2: 12
3: -
4: -
5: -
6: -
7: -
8: -
9: -
```

`22` を入れると衝突する。

```text
2: 12 -> 22
```

`32` を入れるとさらに伸びる。

```text
2: 12 -> 22 -> 32
```

## 実装ポイント
- ハッシュ関数は偏りが少ないほどよい。
- 衝突処理は連結法か開番地法を使う。
- 連結法では、同じ添字にリストをぶら下げる。
- テーブルの大きさと再配置の設計が性能に効く。

## 計算量
| 操作 | 平均 | 最悪 |
| --- | --- | --- |
| 挿入 | O(1) | O(n) |
| 検索 | O(1) | O(n) |
| 削除 | O(1) | O(n) |
